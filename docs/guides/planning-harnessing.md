# 기획 하네싱 가이드

## 목적

기획 에이전트는 아이디어를 대신 결정하는 도구가 아니라, 요구사항, 일정, 리스크, 부서별 산출물을 빠르게 정리하는 도구다.

## 적용 영역

- 국가과제 요구사항 분해
- 회의록 정리와 액션 아이템 추출
- 기능 정의서 초안
- 부서별 업무 분장
- 일정표 초안
- 리스크 목록
- 보고서 초안

## 기획 Task 입력

- 과제명 또는 기능명
- 배경과 목적
- 관련 부서
- 반드시 포함할 산출물
- 마감일
- 제약 조건
- 참고 문서
- 최종 검토자

## 산출물 구조

| 산출물 | 내용 |
|---|---|
| 요구사항표 | 요구사항, 담당 부서, 근거, 완료 기준 |
| 업무분장표 | 역할, 담당자, 입력 자료, 출력 산출물 |
| 일정 초안 | 단계, 시작일, 종료일, 의존관계 |
| 리스크 목록 | 위험, 영향, 대응, 담당 |
| 회의록 요약 | 결정사항, 액션 아이템, 보류사항 |

## 검증 질문

- 요구사항이 실행 가능한 Task로 쪼개졌는가
- 담당자와 산출물이 연결되어 있는가
- 검증 기준이 있는가
- 일정상 의존관계가 빠지지 않았는가
- 외부 제출 문서는 최종 사람이 검토했는가

## 추천 운영

처음에는 "회의록 → 액션 아이템", "요구사항 → 부서별 Task", "QA 결과 → 보고서 문장" 세 가지부터 시작한다.
