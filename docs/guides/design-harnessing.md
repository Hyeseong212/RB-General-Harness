# 디자인 하네싱 가이드

## 목적

디자인 에이전트는 예쁜 시안을 만드는 도구라기보다, 사용자 흐름, 화면 구조, 상태, 문구, 검증 기준을 빠르게 정리하는 역할에 가깝다.

## 적용 영역

- 관리자 화면 레이아웃 정리
- 제품/서비스 소개 자료 구성
- 기능별 UI 상태 정의
- 버튼, 입력, 테이블, 필터, 모달 동작 정리
- 기존 화면과 신규 화면의 차이 비교
- 발표/보고 자료 시각 구조 초안

## 디자인 Task 입력

- 대상 사용자
- 화면 목적
- 기존 화면 또는 참고 자료
- 필수 정보
- 사용자가 해야 하는 행동
- 금지할 톤 또는 스타일
- 모바일/데스크톱 우선순위
- 검증할 상태

## 산출물

- 화면 구조안
- 정보 우선순위
- UI 상태 목록
- 문구 초안
- 사용 흐름
- QA 체크리스트

## 검증 기준

- 사용자가 첫 화면에서 해야 할 일을 바로 알 수 있는가
- 버튼, 입력, 필터, 상태값이 빠짐없이 정의됐는가
- 화면 문구가 실제 업무 용어와 맞는가
- 테이블, 폼, 상세 화면의 정보 밀도가 업무에 맞는가
- 기존 시스템과 비교해야 하는 경우 차이가 문서화됐는가

## 주의점

업무용 관리자 화면은 마케팅 페이지처럼 만들지 않는다.  
반복 사용자가 빠르게 찾고, 비교하고, 수정할 수 있는 구조가 우선이다.
