# Specifications

## User Story

A __user story__ consist of a predefined set of _actions_ taken by a _user_ as a _role_, for a _reason_.

## Storyboard

A __storyboard__ is the steps that a user takes when performing a specific user story scenario. This consists of:

1. Description of the setting of the scenario:
  1. Which user?
  2. Which role?
  3. What is the end goal that the user is trying to achieve?
2. Normal sequence of steps that a user takes.
3. Alternate sequence of steps that a user take for alternate outcome(s), if any.

## Wireframe

A __wireframe__ is a simple markup that a user interface should look like and should minimally consist of interactable elements. It should have a title describing what wireframe is this for.

There will also be simplisitic flow on which UI elements will lead on to the next wireframe.

For ambiguous UI elements present in the wireframe, a description on that element is preferred.

## Combined Storyboard and Wireframe

A __combined storyboard and wireframe__ has some additional elements in addition to what is above.

With reference to steps taken in the storyboard, the wireframe must state the behavior that is to be observe upon each step is taken, if any.

For example, if the button turns green, then the wireframe must annotate that on step X, the button will turn green.
