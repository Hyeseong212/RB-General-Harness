# 회사 역할별 에이전트 적용 맵

## 운영 관점

우리 조직은 대규모 전담 AI 조직을 따로 두기보다, 기존 역할별 반복 업무를 에이전트로 보조하는 방식이 현실적이다.

권장 구조는 다음과 같다.

- PM 또는 소프트웨어 리드가 `전체관리자 에이전트` 역할을 맡는다.
- 각 실무자는 자기 업무의 입력 자료와 판단 기준을 제공한다.
- 에이전트는 초안, 정리, 검증표, 코드 보조, 리스크 목록을 만든다.
- 최종 결정은 담당자가 승인한다.

## 역할별 적용 후보

| 역할 | 인원 | 우선 적용 에이전트 | 주요 Task | 산출물 |
|---|---:|---|---|---|
| 기획자 | 1 | 요구사항 정리 에이전트, 보고서 초안 에이전트 | 국가과제 요구사항 분해, 회의록 액션 아이템 추출, 기능 정의서 초안 | 요구사항표, 기능 명세 초안, 일정/리스크 목록 |
| 소프트웨어 개발자 | 6 | 개발 에이전트, 코드리뷰 에이전트, QA 에이전트 | 레거시 분석, API 설계, 화면 구현, 테스트 케이스 작성, 리팩토링 | 코드 변경안, API 명세, 테스트 결과, 이슈 목록 |
| 하드웨어 조립 | 1 | 작업표준서 에이전트, 검사 체크리스트 에이전트 | 조립 순서 정리, 불량 유형 기록, 검사 항목 표준화 | 조립 매뉴얼, 체크리스트, 불량 로그 요약 |
| F/W | 2 | 펌웨어 분석 에이전트, 프로토콜 검증 에이전트 | I/O 맵 정리, 통신 프로토콜 문서화, 오류 코드 정리, 테스트 로그 분석 | I/O 표, 프로토콜 표, 오류 코드표, 테스트 리포트 |
| 회로 | 1 | 회로 검토 에이전트, BOM 정리 에이전트 | 부품 변경 영향 분석, 전원/신호 경로 설명, BOM 비교 | BOM 차이표, 변경 영향표, 검토 체크리스트 |
| 경영지원 | 1 | 과제비 정리 에이전트, 구매/정산 문서 에이전트 | 과제비 항목 분류, 구매 자료 정리, 정산 증빙 체크 | 비용표, 증빙 체크리스트, 누락 자료 목록 |
| PM | 1 | 전체관리자 에이전트, 리스크 관리 에이전트 | 일정/업무 분배, 이슈 우선순위, 부서 간 handoff | 주간 상태표, 리스크 목록, 의사결정 로그 |
| 재활박사 | 1 | 임상/재활 지식 정리 에이전트 | 훈련 지표 설명, 안전 제한 정리, 효과 리포트 초안 검토 | 의학적 검토 메모, 안전 기준표, 효과 리포트 초안 |

## 추천 시작 순서

1. 회의록을 Task 목록으로 바꾸는 작업
2. 국가과제 요구사항을 부서별 산출물로 나누는 작업
3. 개발/QA 결과를 보고서 문장으로 바꾸는 작업
4. 반복 검증 체크리스트를 만드는 작업
5. 부서 간 handoff 문서를 표준화하는 작업

## 사람 승인 게이트

다음 항목은 에이전트 산출물을 그대로 확정하지 않는다.

- 개인정보, 의료정보, 환자 데이터
- 운영 DB 변경
- 인허가, 임상, 안전성 관련 판단
- 외부 제출 보고서의 최종 문구
- 회로/펌웨어 변경의 안전 영향
- 비용 집행 및 정산 확정
