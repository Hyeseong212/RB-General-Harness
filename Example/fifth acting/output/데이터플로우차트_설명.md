## 데이터 흐름 요약

| 단계 | 생성 데이터 | 저장/조회 |
| --- | --- | --- |
| P-01 운동 목록 | `GET /exercises` | `Exercise` 조회 |
| P-02 시작 | `POST /sessions` | `Session`, `EventLog` write |
| P-03 수행 | `POST /event-logs` | `timer_tick`, `movement_tick` write |
| P-04B 중단 입력 | `POST /safety-inputs` | `SafetyInput`, `EventLog` write |
| P-05 완료 | `PATCH /sessions/{id}` | `Session.state=COMPLETE`, `EventLog` write |
| P-05 완료 요약 | `GET /sessions/{id}/summary` | `Session`, `EventLog`, `SafetyInput` read |
| T-01 치료사 화면 | `GET /sessions/{id}/detail` | `Session`, `EventLog`, `SafetyInput`, `TherapistMemo` read |
| T-01 메모 저장 | `POST /therapist-memos` | `TherapistMemo` write |

## 핵심 메시지

- 환자 화면은 단순 화면 전환이 아니라 이벤트와 상태를 생성한다.
- 치료사 화면은 별도 입력 화면이 아니라, `sessionId` 기준으로 저장된 세션·이벤트·안전 입력을 조회하는 화면이다.
- 중단 입력은 별도 안전 데이터로 분리해야 나중에 전문가 검토와 기록 확인이 가능하다.
- 센서값과 피드백 문구는 실제 제품 기능과 재활 전문가 확인 후 확정한다.

## 주의사항

- 치료, 회복, 효과 입증처럼 읽히는 데이터 라벨은 사용하지 않는다.
- `movement_tick.value`의 단위와 의미는 실제 이승로봇 센서 스펙 확인이 필요하다.
- `pain`, `fatigue`, `dizziness` 허용값은 재활 전문가 확인이 필요하다.
- 차트는 구현 방향 샘플이며 실제 DB 스키마 확정본이 아니다.
