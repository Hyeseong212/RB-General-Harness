# 캡처 QA 하네스

PPT에 들어갈 캡처 이미지는 단순히 예쁘게 보이는 것보다 "무엇을 증명하는 화면인지"가 중요합니다.

## 사용 상황

- 실제 구현 결과를 발표자료에 넣을 때
- 작업 전후 비교 화면을 만들 때
- 관리자 화면, 리포트 화면, 파일 산출물 화면을 보여줄 때
- 민감 정보를 마스킹해야 할 때
- 같은 스타일의 캡처 카드가 여러 장 필요할 때

## 입력 자료

- 원본 캡처 이미지
- 넣을 PPT 페이지 번호
- 보여주고 싶은 메시지
- 마스킹 필요한 영역
- 기존 PPT 스타일

## 작업 절차

1. 캡처 목적을 한 문장으로 적는다.
2. 민감 정보 영역을 먼저 확인한다.
3. 화면에서 강조할 부분을 정한다.
4. PPT 스타일에 맞는 여백, 라벨, 설명 문구를 정한다.
5. 최종 캡처 목록과 페이지별 배치안을 만든다.

## 산출물 형식

```md
## 캡처 목적
## 사용할 이미지 목록
## 페이지별 배치안
## 마스킹 필요 영역
## 강조 포인트
## PPT 문구
## WARN 항목
```

## 캡처 검증표

| 항목 | 확인 |
| --- | --- |
| 실제 구현 화면인가 | 예/아니오 |
| 발표 메시지와 직접 연결되는가 | 예/아니오 |
| 환자, 계정, 운영 정보가 마스킹되었는가 | 예/아니오 |
| 이미지 비율이 PPT에 맞는가 | 예/아니오 |
| 텍스트가 너무 작지 않은가 | 예/아니오 |
| 같은 페이지의 다른 요소와 겹치지 않는가 | 예/아니오 |

## 좋은 캡처의 기준

- 한 장의 캡처가 하나의 메시지만 말한다.
- UI 전체를 보여주되 핵심 영역은 명확히 보인다.
- 캡처 위에 긴 설명을 얹지 않는다.
- 실제 구현 산출물이라는 맥락이 보인다.
- 민감 정보는 원본 단계에서 마스킹한다.

## 사람 승인 게이트

환자 정보, 계정 정보, 운영 데이터, 제품 성능을 암시하는 화면은 발표자료에 넣기 전에 PM 또는 담당자가 확인합니다.
