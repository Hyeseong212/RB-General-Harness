# PPT 디자인 레퍼런스 사이트 및 시스템

## 목적

이 문서는 특정 브랜드 템플릿을 고정하기 위한 문서가 아니다.  
PPT 제작 에이전트가 참고할 수 있는 디자인 레퍼런스의 종류와 사용 기준을 정리한 범용 하네스다.

## 레퍼런스 사용 원칙

- 레퍼런스는 그대로 베끼는 대상이 아니라 구조, 밀도, 정보 배치, 시각화 방식을 참고하는 자료다.
- 발표 목적, 청중, 전달 메시지가 먼저 정해져야 한다.
- 회사 로고, 컬러, 폰트, 표기 규칙은 프로젝트별 별도 디자인 시스템 문서가 있으면 그 문서를 우선한다.
- 레퍼런스가 없을 때도 16:9, 한 장 한 메시지, 일관된 여백과 폰트 크기를 지킨다.

## 추천 레퍼런스 유형

| 유형 | 참고 목적 | 확인할 것 |
|---|---|---|
| SaaS 제품 소개 자료 | 복잡한 기능을 간단히 설명하는 방식 | 문제-해결-효과 흐름, 기능 캡처 배치 |
| 컨설팅 보고서 | 수치, 비교, 전략 구조화 | 표, 2x2 매트릭스, 로드맵, 핵심 메시지 |
| 투자/사업계획서 | 시장, 비용, 일정, 팀 구성 설명 | 스토리라인, 지표 카드, 비용 구조 |
| 기술 제안서 | 아키텍처와 구현 계획 설명 | 시스템 다이어그램, 단계별 구축안 |
| 제품 데모 자료 | 실제 작동 화면 설명 | 캡처 순서, Before/After, 사용 흐름 |

## 에이전트에 줄 입력 자료

```text
발표 목적:
청중:
핵심 메시지:
슬라이드 수:
사용할 브랜드 기준:
참고하고 싶은 레퍼런스 링크:
피해야 할 스타일:
반드시 들어갈 실제 자료:
완료 기준:
```

## 레퍼런스 분석 요청 예시

```text
아래 레퍼런스를 그대로 복제하지 말고, 정보 배치 방식만 분석해줘.
분석 결과는 다음 형식으로 정리해줘.

1. 슬라이드 구조
2. 제목 작성 방식
3. 본문 정보 밀도
4. 색상/타이포그래피 특징
5. 우리 발표자료에 적용 가능한 점
6. 적용하면 안 되는 점
```

## PPT 제작 요청 예시

```text
아래 내용을 16:9 발표자료 구조로 바꿔줘.
한 장에는 하나의 메시지만 두고, 실제 화면 캡처나 표가 들어갈 자리도 표시해줘.
각 슬라이드는 제목, 핵심 메시지, 시각화 아이디어, 필요한 자료, 발표자 메모를 포함해줘.
```

## 디자인 QA 체크리스트

- 제목만 읽어도 각 장의 결론이 보이는가
- 한 장에 메시지가 하나만 있는가
- 실제 화면, 표, 수치가 들어갈 자리가 명확한가
- 장식보다 정보 구조가 우선되는가
- 폰트 크기와 여백이 전체 장표에서 일관되는가
- 외부 레퍼런스의 브랜드 요소를 그대로 가져오지 않았는가
- 영상이 들어가는 장표라면 영상 자체가 주인공이고, 주변 설명은 최소화되어 있는가
