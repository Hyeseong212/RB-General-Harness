# 구매 견적 비교 하네스

## 목적

여러 견적서와 구매 요청을 같은 기준으로 비교해 구매 담당자와 PM이 빠르게 판단할 수 있는 표를 만든다.

## 입력 자료

- 견적서 원본
- 구매 요청 사유
- 필요한 수량과 납기
- 예산 한도
- 기존 거래처 정보
- 필수 조건: 보증, 설치, 교육, 유지보수, 배송비, VAT

## 에이전트 역할

- 견적서의 항목명을 통일한다.
- 단가, 수량, 총액, VAT, 배송비를 분리한다.
- 납기, 결제 조건, 보증 기간, 유지보수 조건을 표로 만든다.
- 비교 기준이 다른 항목은 WARN으로 표시한다.
- 구매 담당자가 업체에 다시 물어볼 질문을 만든다.

## 산출물 형식

```md
## 구매 요청 요약
## 견적 비교표
## 총액 비교
## 조건 비교
## 누락 또는 확인 필요 항목
## 추천 검토 방향
## 승인 필요 항목
```

| B |  |  |  | 포함/별도 |  |  |  |  |  |

## 검증 기준

- VAT 포함 여부가 명확한가
- 배송비와 설치비가 분리되어 있는가
- 납기와 결제 조건이 있는가
- 동일 품목 또는 동일 사양 기준으로 비교했는가
- 사양이 다르면 어떤 차이인지 표시했는가

## 사람 승인 게이트

최종 업체 선정, 가격 협상, 발주, 결제, 선금 지급은 사람이 승인합니다.
