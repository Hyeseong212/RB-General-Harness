# 개발 폴더

소프트웨어 개발자가 코드 분석, 구현, QA, 인계를 진행할 때 사용하는 하네스 폴더입니다. 핵심은 많이 만들기보다 작게 바꾸고, 검증 가능한 결과를 남기는 것입니다.

## 이 폴더를 쓰는 경우

- 기존 코드 구조를 파악하고 변경 범위를 정해야 할 때
- 버그 수정, 리팩토링, 기능 추가를 작은 Task로 나눌 때
- 컨텍스트가 큰 작업에서 서브 에이전트를 나눠야 할 때
- 테스트, 빌드, 실행 확인 결과를 남겨야 할 때
- 다음 개발자에게 변경 내용을 인계해야 할 때

## 문서 구성

- `01_개발_에이전트_운영원칙.md`: 개발 업무에서 에이전트를 쓰는 기본 원칙
- `02_카파시_코딩_가이드라인.md`: 보수적이고 검증 중심인 코딩 원칙
- `03_서브에이전트_생성기준.md`: 대용량 컨텍스트와 모듈 분리 기준
- `04_개발_Task_브리프_템플릿.md`: 개발 작업을 맡길 때 쓰는 입력 양식

## 개발 하네스 기본 원칙

- 구현 전에 목적, 입력, 출력, 검증 기준을 먼저 확인한다.
- 요청과 직접 관련 없는 리팩토링은 하지 않는다.
- 기존 코드 스타일과 구조를 따른다.
- 변경한 내용은 테스트, 빌드, 실행 확인 중 가능한 방식으로 검증한다.
- 검증하지 못한 부분은 숨기지 않고 WARN으로 남긴다.

## 서브 에이전트 사용 기준

- 코드베이스가 커서 한 번에 맥락을 잡기 어려울 때 탐색 역할을 분리한다.
- 프론트엔드, 백엔드, DB, 테스트처럼 파일 소유 범위가 분리될 때 병렬 작업을 맡긴다.
- 같은 파일을 여러 에이전트가 동시에 수정하게 하지 않는다.
- 최종 통합자는 변경 파일, 검증 결과, 남은 리스크를 확인한다.
