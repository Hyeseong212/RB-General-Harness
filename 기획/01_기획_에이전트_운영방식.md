# 기획 에이전트 운영방식

## 목적

기획 에이전트는 의사결정을 대신하는 도구가 아니다.  
회의, 요구사항, 아이디어, 일정, 리스크를 정리 가능한 산출물로 바꾸는 도구다.

## 주요 역할

| 역할 | 설명 |
|---|---|
| 브레인스토밍 에이전트 | 가능한 아이디어와 적용 후보를 넓게 뽑는다. |
| 정리 에이전트 | 비슷한 내용을 묶고 중복을 제거한다. |
| 문서화 에이전트 | 회의록, 요구사항, 기획서 초안을 만든다. |
| 산출물 변환 에이전트 | PPT, MD, 표, XML 같은 형식으로 변환한다. |
| 리스크 점검 에이전트 | 빠진 이해관계자, 일정 리스크, 검증 항목을 찾는다. |

## 기획자가 쓰기 좋은 업무

- 회의록을 액션 아이템으로 바꾸기
- 국가과제 요구사항을 부서별 Task로 나누기
- 아이디어를 기능 후보로 정리하기
- 보고서 목차 만들기
- PPT 기획서 흐름 만들기
- MD 문서 초안 작성
- XML 또는 구조화된 문서 형식 초안 작성
- 일정과 리스크 목록 만들기

## 기획자가 직접 판단해야 하는 일

- 최종 우선순위
- 고객/기관 제출 문구
- 일정 확정
- 예산과 인력 배정
- 제품 방향성
- 의료/임상/인허가 관련 최종 판단

## 운영 순서

1. 브레인스토밍으로 후보를 넓게 뽑는다.
2. 같은 맥락끼리 묶는다.
3. 실제 실행 가능한 Task로 바꾼다.
4. 문서 형식을 정한다.
5. 사람이 최종 검토하고 확정한다.
