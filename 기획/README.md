# 기획 폴더

기획자와 PM이 회의록, 요구사항, PPT 초안, 제안서 문구를 정리할 때 사용하는 하네스 폴더입니다. 에이전트는 기획 결정을 대신하지 않고, 흩어진 정보를 구조화해서 사람이 검토하기 쉬운 초안을 만드는 역할을 맡습니다.

## 이 폴더를 쓰는 경우

- 회의록에서 결정 사항, 액션 아이템, 질문 목록을 뽑을 때
- 요구사항을 기능, 화면, 데이터, 정책, 리스크로 나눌 때
- PPT 페이지별 메시지와 목차를 만들 때
- Claude, GPT 등을 사용해 기획서 초안을 만들고 정리할 때
- 개발, 디자인, 하드웨어 쪽으로 넘길 인계 문서를 만들 때

## 문서 구성

- `01_기획_에이전트_운영방식.md`: 기획 업무에서 에이전트를 쓰는 기본 방식
- `02_브레인스토밍_에이전트.md`: 아이디어 발산과 비교 정리용
- `03_정리_문서화_에이전트.md`: 회의록, 요구사항, 보고 문서 정리용
- `04_기획서_PPT_MD_XML_산출물.md`: PPT, MD, XML 등 산출물화 방식

## 검증 기준

- 결정된 내용과 가정한 내용이 분리되어 있는가
- 담당자, 일정, 다음 액션이 빠지지 않았는가
- 외부 제출 문구가 내부 의견처럼 작성되어 있지 않은가
- 다른 부서가 바로 이어받을 수 있을 정도로 입력 자료가 충분한가

## 사람 승인 게이트

과제 제출 문구, 대외 발표 문구, 계약 관련 표현, 제품 성능을 단정하는 표현은 기획자와 PM이 반드시 확인합니다.
