# PPT 반영 인수인계

## 작업 히스토리

- 작성 의도: PPT 파일을 직접 수정하는 작업이 아니라, 이후 PPT 작업자가 9페이지 이후에 내용을 써넣을 수 있도록 남기는 운영 예시 메모다.
- 반영 대상: `Example` 폴더에서 세팅한 자료조사 하네스 흐름
- 표현 기준: 실제 실행 결과처럼 보이지 않게 하고, "에이전트를 자료조사 담당으로 세팅하는 방식"을 보여준다.
- 주의: 긴 요청문을 장표에 그대로 넣지 않는다. 역할, 입력 하네스, first acting 산출물, second acting 인계 구조, 표현 분리 이유만 시각화한다.

## 정정 히스토리

- 2026-05-12 정정: second acting에서 다루는 PPT는 기존 발표자료 9페이지 이후에 삽입하는 장표가 아니다.
- 정정된 해석: first acting 결과를 바탕으로 별도의 신규 "이승로봇 게임재활 콘텐츠 기획 PPT"를 만드는 단계다.
- 따라서 이 파일의 9페이지 이후 반영 메모는 기존 하네싱 발표자료 보강 아이디어로만 남기고, 실제 second acting 산출물은 `second acting/output/신규기획PPT_구성안.md`와 `second acting/output/신규기획PPT_장표문구_초안.md`를 기준으로 본다.
- 추가 메모: 기존 발표자료 9페이지 이후에 내용을 넣는 작업은 나중에 할 일이다. 현재는 해당 작업을 진행하지 않고 히스토리만 남긴다.

## 반영 위치

현재 발표자료의 9페이지 이후에 이 내용을 추가한다.

기존 흐름을 끊지 않도록, 9페이지 이후에 `Example` 폴더에서 세팅한 자료조사 하네스 운영 흐름을 2~3페이지 정도로 자연스럽게 삽입한다.

## PPT에 넣을 핵심 내용

이번 내용은 실제 실행 결과를 보여주는 장표가 아니라, 하네스를 사용해 에이전트를 어떻게 세팅하고 업무를 나누는지 보여주는 운영 예시다.

예시 주제:

- 이승로봇 콘텐츠 기획

운영 방식:

1. 자료조사 하네스를 먼저 읽게 한다.
2. 브레인스토밍, 자료조사, 재활의학 관점을 first acting에서 묶는다.
3. first acting의 결과를 second acting으로 넘겨 PPT 디자인 하네스와 연결한다.
4. UI/UX가 필요하면 third acting으로 분리한다.

## PPT에서 강조할 메시지

하네스는 단순한 문서 모음이 아니라, 에이전트에게 업무 역할과 검증 기준을 부여하는 세팅 파일이다.

자료조사 에이전트는 PPT를 바로 만드는 역할이 아니다.  
먼저 출처, 안전하게 쓸 수 있는 문장, 위험한 문장, 전문가 확인 질문을 정리하는 역할을 맡는다.

이렇게 나누면 다음 단계의 PPT 제작 에이전트가 조사 내용을 그대로 복사하지 않고, 검증된 문장과 구조만 가져가 발표자료로 바꿀 수 있다.

## PPT에 넣을 수 있는 장표 구성안

### 추가 슬라이드 1. 자료조사 에이전트는 먼저 근거와 위험 표현을 정리한다

넣을 내용:

- 읽을 하네스: `advanced-task-brief`, `리서치_의사결정`, `팩트체킹`, `재활근거_팩트체킹`
- 역할: 자료조사 담당
- 산출물: 조사 요약, 팩트체킹 로그, 콘텐츠 기획 재료, 전문가 확인 질문

표현 방향:

- “PPT를 바로 만들지 않고, 근거와 표현 안전성을 먼저 분리한다.”

### 추가 슬라이드 2. first acting과 second acting을 나누면 작업이 덜 꼬인다

넣을 내용:

- first acting: 브레인스토밍, 자료조사, 재활의학 관점 정리
- second acting: PPT 디자인 하네스와 연결해 장표화
- 선택 third acting: UI/UX 화면 흐름으로 확장

표현 방향:

- “한 에이전트에게 모든 일을 맡기지 않고, Act별로 역할과 산출물을 나눈다.”

### 추가 슬라이드 3. 자료조사 산출물은 다음 에이전트의 입력값이 된다

넣을 내용:

- 안전한 문장
- 위험한 문장
- 출처 URL
- 확인일
- 전문가 확인 질문
- PPT 제작용 핵심 메시지

표현 방향:

- “자료조사 결과는 최종 문서가 아니라 다음 작업자를 위한 검증된 입력값이다.”

## PPT 스타일 메모

- 기존 발표자료 톤과 맞춰서 꾸민다.
- 실행 결과처럼 과장하지 않고, 운영 방식 예시로 표현한다.
- 긴 요청문을 그대로 붙이지 말고, 역할/입력/산출물/검증 기준 중심으로 시각화한다.
- 실제 파일 경로는 작은 주석 또는 하단 보조 정보로만 넣는다.
- 핵심 메시지는 슬라이드 제목만 읽어도 이해되게 쓴다.

## 다음 PPT 작업자에게 전달할 문장

```text
9페이지 이후에 Example 폴더의 자료조사 하네스 세팅 내용을 추가해줘.

이 내용은 실제 실행 결과가 아니라, 에이전트를 자료조사 담당으로 세팅하는 운영 예시야.
PPT에는 긴 요청문을 그대로 넣지 말고, 다음 구조로 꾸며서 넣어줘.

1. 자료조사 에이전트가 먼저 읽는 하네스
2. first acting에서 정리할 산출물
3. second acting으로 넘기는 인계 구조
4. 안전한 문장, 위험한 문장, 출처, 전문가 확인 질문을 분리하는 이유

핵심 메시지:
자료조사 에이전트는 PPT를 바로 만드는 사람이 아니라, 다음 에이전트가 믿고 쓸 수 있는 근거와 표현 기준을 정리하는 역할이다.
```
