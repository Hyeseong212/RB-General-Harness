# 하드웨어 FW 회로 폴더

하드웨어 조립, F/W, 회로 담당자가 변경 영향과 검증 항목을 정리할 때 사용하는 하네스 폴더입니다. 에이전트는 실제 장비 검증을 대신하지 않고, 조립 체크리스트, 회로 변경 영향, 펌웨어 릴리즈 체크, 소프트웨어 인계 문서를 만드는 역할을 맡습니다.

## 이 폴더를 쓰는 경우

- 센서, 모터, 보드, 케이블, 회로 변경 영향을 정리할 때
- F/W 변경이 앱, 서버, 데이터 포맷에 미치는 영향을 볼 때
- 펌웨어 버전, 프로토콜, 로그 포맷을 소프트웨어팀에 인계할 때
- 장비 조립 절차와 QC 항목을 표준화할 때
- 회로 변경에 따른 전원, 보호회로, 커넥터, 핀맵 영향을 정리할 때
- 통합 테스트에서 발생한 현장 로그와 재현 조건을 문서화할 때

## 문서 구성

- `01_HW_FW_회로_변경영향_하네스.md`: HW/FW/회로 변경 영향 공통 정리
- `02_하드웨어_조립_QC_하네스.md`: 조립 절차, 부품, 케이블, QC 체크
- `03_FW_릴리즈_검증_하네스.md`: 펌웨어 릴리즈, 버전, 프로토콜, 로그 검증
- `04_회로_변경_ECN_하네스.md`: 회로 변경, 핀맵, BOM, 변경 승인 정리
- `05_통합테스트_현장로그_하네스.md`: 장비-앱-서버 통합 테스트와 현장 로그 정리
- `06_SW_HW_FW_인계_하네스.md`: 소프트웨어팀과 하드웨어/FW팀 간 인계 문서

## 기본 입력 자료

- 장비 사진, 조립 사진, 부품 목록, BOM
- 회로도, 핀맵, 커넥터 정보, 배선표
- 펌웨어 버전, 프로토콜 문서, 통신 로그
- 테스트 조건, 재현 절차, 실패 증상
- 소프트웨어팀에 전달해야 할 제약 조건

## 검증 기준

- 변경 전후 차이가 명확히 적혀 있는가
- HW, FW, 회로, 앱, 서버 영향이 분리되어 있는가
- 실제 장비에서 확인해야 할 항목이 따로 표시되어 있는가
- 로그, 펌웨어 버전, 테스트 조건이 남아 있는가
- 배포 또는 출고 전 사람 승인 게이트가 표시되어 있는가

## 사람 승인 게이트

전기적 안전, 의료기기 동작, 펌웨어 배포, 회로 변경, 장비 출고와 관련된 내용은 담당 엔지니어가 직접 검증하고 승인합니다. 에이전트는 체크리스트와 인계 문서를 만들 수 있지만 실제 장비 안전성을 확정하지 않습니다.

## HW/FW 하네스에서 특히 조심할 점

- 실제 측정값 없이 정상 동작을 확정하지 않는다.
- 펌웨어 버전과 장비 버전을 항상 같이 기록한다.
- 회로 변경은 변경 사유, 영향 범위, 승인자를 남긴다.
- 소프트웨어팀에 넘길 때 프로토콜, 로그 예시, 오류 코드, 재현 절차를 함께 제공한다.
