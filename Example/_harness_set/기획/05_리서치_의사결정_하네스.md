# 리서치 의사결정 하네스

기획 업무에서 에이전트는 결정을 대신하지 않습니다. 대신 자료를 모으고, 선택지를 비교하고, 결정자가 볼 수 있는 판단표를 만듭니다.

## 사용 상황

- 가격 정책 비교
- 제품 기능 우선순위 정리
- PPT 목차 결정
- 외부 발표 문구 검토
- 경쟁사 또는 레퍼런스 조사
- 부서별 운영 방식 정리

## 입력 자료

- 현재 요구사항
- 회의록
- 기존 PPT 또는 기획서
- 비교 대상 목록
- 결정 기준
- 반드시 제외할 선택지

## 작업 절차

1. 문제를 한 문장으로 다시 정의한다.
2. 결정 기준을 3~5개로 나눈다.
3. 선택지를 표로 비교한다.
4. 근거가 부족한 항목은 WARN으로 표시한다.
5. 추천안을 1안, 2안, 보류안으로 제시한다.
6. 사람 결정이 필요한 질문을 남긴다.

## 산출물 형식

```md
## 결정해야 할 문제
## 선택지
## 비교 기준
## 비교표
## 추천안
## WARN 항목
## 최종 결정자 질문
## PPT에 넣을 수 있는 한 문장
```

## 비교표 예시

| 선택지 | 장점 | 단점 | 비용 | 리스크 | 추천도 |
| --- | --- | --- | --- | --- | --- |
| A안 |  |  |  |  | 높음/중간/낮음 |
| B안 |  |  |  |  | 높음/중간/낮음 |
| 보류 |  |  |  |  | 높음/중간/낮음 |

## 좋은 기획 하네스 기준

- 근거와 의견이 섞이지 않는다.
- 추천안이 하나만 있지 않고 대안이 있다.
- 비용, 일정, 리스크가 따로 보인다.
- PPT에 바로 옮길 수 있는 문장이 있다.
- 결정자가 답해야 할 질문이 남는다.

## 사람 승인 게이트

- 외부 제출 문구
- 가격 정책
- 계약 또는 비용 결정
- 의료적 성능 표현
- 회사의 공식 입장으로 보일 수 있는 문장
