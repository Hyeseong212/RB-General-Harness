# 디자인 에이전트 운영방식

## 목적

디자인 에이전트는 단순히 예쁜 이미지를 만드는 도구가 아니다.  
화면 목적, 사용자 흐름, 정보 우선순위, 상태, 문구를 정리하는 도구다.

## 적용 영역

- 관리자 UI 구조 정리
- 화면별 정보 우선순위 정의
- 버튼, 입력, 필터, 테이블, 모달 상태 정리
- 기존 화면과 신규 화면 비교
- PPT 시각 구조 초안
- 디자인 QA 체크리스트 작성

## 디자인 에이전트 입력

- 대상 사용자
- 화면 목적
- 사용자가 해야 하는 행동
- 필수 정보
- 참고 화면
- 금지할 스타일
- 데스크톱/모바일 우선순위
- 필요한 상태값

## 출력 산출물

- 화면 구성안
- 사용자 흐름
- 컴포넌트 목록
- 상태별 문구
- QA 체크리스트
- PPT 슬라이드 구조안

## 주의점

업무용 화면은 마케팅 페이지처럼 만들지 않는다.  
반복 사용자가 빠르게 찾고, 비교하고, 수정할 수 있는 구조가 우선이다.

PPT는 예쁘게 꾸미는 것보다 한 장에 하나의 메시지가 명확히 보이는 것이 우선이다.
