# 팩트체킹 법률 규제 하네스

법률, 규제, 의료기기 인허가, 개인정보, 정부지원사업, 가격 정책처럼 바뀔 수 있는 정보는 에이전트의 기억만으로 답하면 안 됩니다. 반드시 인터넷에서 공식 출처를 실제 확인하고, 확인 근거를 남깁니다.

## 반드시 인터넷 확인이 필요한 경우

- 법률, 시행령, 시행규칙, 고시, 행정규칙
- 의료기기 인허가, GMP, 사이버보안, SaMD, 생성형 AI 의료기기 가이드라인
- 개인정보보호법, 가명정보, 환자/계정 정보 처리
- 정부지원사업 공고, 과제비 집행 기준, 정산 규정
- 가격 정책, 구독 요금, 라이선스 조건
- 세무, 노무, 계약, 표시광고, 약관, 소비자 보호
- 외부 발표자료에 들어가는 규제 또는 법률 문구

## 공식 출처 우선순위

| 분야 | 1순위 공식 출처 | 보조 출처 |
| --- | --- | --- |
| 국내 법령 | 국가법령정보센터 `law.go.kr` | 소관 부처 법령 페이지 |
| 의료기기 | 식품의약품안전처 `mfds.go.kr` | 의료기기전자민원창구, 한국의료기기안전정보원 |
| 개인정보 | 개인정보보호위원회 `pipc.go.kr`, 개인정보포털 `privacy.go.kr` | KISA, 관계부처 가이드라인 |
| 공정거래/표시광고 | 공정거래위원회 `ftc.go.kr` | 국가법령정보센터 |
| 정부지원사업 | 주관 부처, 전담기관, 사업 공고 원문 | 기업마당, 중소벤처24 |
| 조달/계약 | 조달청, 나라장터, 국가법령정보센터 | 발주기관 공고문 |
| 가격 정책 | 서비스 제공사의 공식 pricing/help 페이지 | 공식 약관, 관리자 콘솔 |

## 확인 절차

1. 질문이 법률, 규제, 가격, 의료, 개인정보, 정부지원사업에 해당하는지 먼저 판단한다.
2. 해당하면 "인터넷 확인 필요"로 표시한다.
3. 공식 출처를 먼저 검색한다.
4. 문서명, 발행기관, 시행일, 개정일, 등록일을 확인한다.
5. 필요한 경우 2개 이상의 공식 또는 준공식 출처로 교차 확인한다.
6. 에이전트 해석과 원문 근거를 분리한다.
7. 법률 자문, 의료 판단, 계약 결정은 사람 승인 게이트로 넘긴다.

## 검색어 예시

```text
site:law.go.kr 의료기기법 시행규칙 최신
site:mfds.go.kr 의료기기 소프트웨어 허가 심사 가이드라인 민원인안내서
site:pipc.go.kr 개인정보 보호 가이드라인 가명정보
site:ftc.go.kr 표시광고법 고시 지침
site:mss.go.kr 정부지원사업 공고 2026
```

## 법률 문구 작성 원칙

- "법적으로 가능하다"처럼 단정하지 않는다.
- "공식 출처 기준으로는 다음 조항/가이드라인 확인이 필요하다"처럼 근거 중심으로 쓴다.
- 법령 원문과 에이전트 해석을 분리한다.
- 시행일과 개정일을 반드시 확인한다.
- 외부 제출자료에는 "전문가 검토 필요" 여부를 표시한다.

## 의료기기 관련 추가 원칙

- 식약처 가이드라인은 등록일, 고시일, 이전 이력을 확인한다.
- 의료기기 소프트웨어, 사이버보안, GMP, 허가·심사 문서는 최신 개정본인지 확인한다.
- 제품 성능, 치료 효과, 환자 안전 문구는 재활 도메인 전문가와 RA 담당자 확인 전 확정하지 않는다.

## 가격 정책 관련 추가 원칙

- 블로그나 커뮤니티가 아니라 공식 pricing/help 페이지를 확인한다.
- 월간/연간, 세금 포함 여부, 최소 좌석, 사용량 제한, 약정 조건을 분리한다.
- "현재 확인일 기준"이라고 표시한다.
- 계약형 Enterprise 가격은 공개가 없으면 추정하지 않고 "영업 문의 필요"로 표시한다.
