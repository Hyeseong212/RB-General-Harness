# UI/UX 화면상세 컴포넌트 액션

- 작성일: 2026-05-12
- 기준 해상도: 모바일 세로 `1080x1920`
- 대상 PPTX: `이승로봇_게임재활_UIUX기획_화면상세_20260512.pptx`
- 목적: 플로우만 전달하지 않고, 화면 안의 배치 위치와 컴포넌트 액션을 개발자가 바로 확인할 수 있게 분리한다.

## 공통 배치 기준

| 항목 | 기준 |
| --- | --- |
| safe margin | 좌우 `96px` |
| header zone | `y=120~260` |
| body zone | `y=320~1480` |
| bottom CTA zone | `y=1600~1780` |
| 안전 액션 | 수행 중 화면에서는 하단 고정 노출 |
| 문구 제한 | 치료, 회복, 효과 입증처럼 제품 효과로 읽히는 문장 금지 |

## 화면별 컴포넌트 액션

### P-01 운동 선택

| 컴포넌트 | 위치 기준 | 사용자 액션 | 이벤트/다음 |
| --- | --- | --- | --- |
| 운동 카드 | `x=96`, `w=888`, 첫 카드 `y=420`, `h=260`, gap `48` | 탭 | `tapExercise -> P-02` |
| 검토 필요 카드 | 카드 목록 안 | 탭 시 차단 | `disabledReason` 표시 |

### P-02 미션 안내

| 컴포넌트 | 위치 기준 | 사용자 액션 | 이벤트/다음 |
| --- | --- | --- | --- |
| 미션 요약 | `x=96`, `y=420` | 확인 | `Mission.durationSec` 표시 |
| 주의 박스 | `x=96`, `y=700` | 읽기 | `safetyCopy` 표시 |
| 시작 버튼 | `y=1550`, `h=120` | 탭 | `tapStart -> P-03` |
| 뒤로 버튼 | `y=1690`, `h=120` | 탭 | `back -> P-01` |

### P-03 미션 수행

| 컴포넌트 | 위치 기준 | 사용자 액션 | 이벤트/다음 |
| --- | --- | --- | --- |
| 타이머 | `x=96`, `y=420` | 자동 감소 | `timer_tick` 저장 |
| 목표 반응 | `x=240`, `y=720` | 동작 감지 시 시각 변화 | `movement_tick` 저장 |
| 쉬어가기 버튼 | `x=96`, `y=1640` | 탭 | `tapRest -> P-04A` |
| 중단 입력 버튼 | `x=576`, `y=1640` | 탭 | `tapStop -> P-04B` |
| 시간 종료 | 자동 | 자동 이동 | `timerEnd -> P-05` |

### P-04A 쉬어가기

| 컴포넌트 | 위치 기준 | 사용자 액션 | 이벤트/다음 |
| --- | --- | --- | --- |
| 휴식 타이머 | `x=180`, `y=520` | 자동 카운트 | `rest_tick` |
| 안내 문구 | `x=96`, `y=1000` | 확인 | `restCopy` 표시 |
| 계속하기 버튼 | `x=96`, `y=1600` | 탭 | `tapResume -> P-03` |
| 중단 입력 버튼 | `x=576`, `y=1600` | 탭 | `tapStop -> P-04B` |

### P-04B 중단 입력

| 컴포넌트 | 위치 기준 | 사용자 액션 | 이벤트/다음 |
| --- | --- | --- | --- |
| pain 라디오 | 필드 시작 `x=96`, `y=380`, `h=96` | `0~3` 선택 | `SafetyInput.pain` |
| fatigue 라디오 | 동일 필드 규칙 | `0~5` 선택 | `SafetyInput.fatigue` |
| dizziness 토글 | 동일 필드 규칙 | 있음/없음 선택 | `SafetyInput.dizziness` |
| reason 셀렉트 | 동일 필드 규칙 | 중단 사유 선택 | `EventLog.reason` |
| 저장하고 종료 | `x=96`, `y=1540` | 탭 | `stop_reason_saved -> P-05` |
| 취소 | `x=96`, `y=1680` | 탭 | `P-04A` 또는 `P-03` 복귀 |
| reason 누락 | 저장 탭 | 오류 | `중단 사유를 선택하세요.` 표시 |

### T-01 치료사 세션 상세

## 개발 티켓 기준

| 티켓 | 구현 단위 | 완료 조건 |
| --- | --- | --- |
| UX-01 | P-01 운동 선택 | `requiresReview` 탭 차단 |
| UX-02 | P-02 안내 화면 | `tapStart` 저장 |
| UX-03 | P-03 수행 화면 | `rest/stop` 버튼 항상 노출 |
| UX-04 | P-04A 쉬어가기 | `REST` 상태 전이 |

## 확인 필요

- 실제 이승로봇 센서 입력값과 화면 반응 방식
- `pain`, `fatigue`, `dizziness` 입력 범위의 적절성
- `쉬어가기`, `중단 입력` 버튼을 모든 수행 중 상태에서 항상 노출할지 여부
- 치료 효과처럼 읽히지 않는 환자 피드백 문구
- 의료기기/광고 관점의 외부 발표 가능 문구
