# UI/UX 기획 PPT 구성안

- 작성일: 2026-05-12
- 단계: third acting
- 대상: 이승로봇 게임재활 UI/UX 기획 PPT
- 목적: second acting의 콘텐츠 기획을 실무 개발자가 구현 가능한 화면, 상태, 기록 데이터, 확인 질문으로 변환한다.

## 한 문장 목표

한 가지 재활운동을 30초 게임 미션으로 만들 때 필요한 환자 화면, 치료사 화면, 상태 전이, 기록 데이터, 개발 범위를 정리한다.

## 발표 대상

- 실무 개발자
- UI/UX 디자이너
- PM
- 재활 전문가/RA 검토 담당자

## PPT 구성

| 페이지 | 제목 | 역할 |
| --- | --- | --- |
| 1 | 30초 미션을 실제 화면으로 옮기는 UI/UX 기획 | third acting 목적 제시 |
| 2 | 이번 PPT의 목적은 화면을 예쁘게 그리는 것이 아니라 개발 단위를 나누는 것이다 | 개발 인계 범위 정의 |
| 3 | 30초 미션의 기본 사용자 흐름 | 전체 사용자 플로우 |
| 4 | 화면 지도는 환자용·치료사용·운영용으로 분리한다 | IA / 화면 목록 |
| 5 | 환자 시작 화면은 선택보다 안심을 먼저 준다 | 운동 선택 화면 |
| 6 | 수행 화면은 목표·반응·휴식 버튼이 한눈에 보여야 한다 | 미션 수행 화면 |
| 7 | 상태 설계는 재미보다 안전 분기를 먼저 잡아야 한다 | 상태/인터랙션 |
| 8 | 완료 화면은 점수판이 아니라 다음 판단을 돕는 요약이다 | 세션 완료 화면 |
| 9 | 치료사 화면은 한 세션의 관찰 근거를 빠르게 보여준다 | 치료사 세션 상세 |
| 10 | 개발자가 필요한 최소 데이터 구조 | 데이터/이벤트 구조 |
| 11 | MVP 범위는 한 운동, 한 미션, 두 화면 묶음으로 시작한다 | 개발 범위 |
| 12 | 개발 인계 전에 남겨야 할 확인 질문 | handoff checklist |

## 핵심 설계 방향

- 환자 화면은 `오늘의 미션`, `수행 중 반응`, `쉬어가기`, `중단 입력`을 우선한다.
- 치료사 화면은 점수보다 `세션 기록`, `관찰 메모`, `다음 세션 참고`를 우선한다.
- 상태는 `READY`, `ACTIVE`, `REST`, `STOPPED`, `COMPLETE`로 나누어 설계한다.
- 기록 데이터는 `Exercise`, `Mission`, `Session`, `EventLog`, `SafetyInput` 단위로 시작한다.
- MVP는 전체 운동 라이브러리가 아니라 `팔 앞으로 뻗기 30초 미션` 하나로 닫히게 만든다.

## 사용하면 안 되는 방향

- 치료 효과, 회복, 기능 향상처럼 읽히는 화면 문구
- 실시간 치료 판정처럼 보이는 UI
- 자동 처방 추천처럼 보이는 기능
- 환자 안전 입력 없이 진행만 강조하는 게임 화면
