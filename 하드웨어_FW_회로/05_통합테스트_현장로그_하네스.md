# 통합테스트 현장로그 하네스

## 목적

장비, 펌웨어, 앱, 서버가 함께 동작하는 테스트에서 재현 조건, 로그, 실패 증상, 담당 영역을 정리한다.

## 입력 자료

- 테스트 일시와 장소
- 장비 버전과 시리얼
- FW 버전
- 앱 버전
- 서버 환경
- 테스트 절차
- 현장 로그와 캡처
- 실패 증상

## 에이전트 역할

- 재현 절차를 단계별로 정리한다.
- 로그와 증상을 시간순으로 배열한다.
- 원인 후보를 HW, FW, 앱, 서버, 사용자 절차로 나눈다.
- 추가 테스트가 필요한 항목을 표시한다.
- 담당자별 액션 아이템을 만든다.

## 현장 로그 표

| 시간 | 조건 | 관찰 증상 | 로그/캡처 | 원인 후보 | 담당 |
| --- | --- | --- | --- | --- | --- |
|  |  |  |  | HW/FW/앱/서버/절차 |  |

## 산출물 형식

```md
## 테스트 환경
## 재현 절차
## 현장 로그
## 원인 후보
## 추가 테스트
## 담당자별 액션
## BLOCKED 항목
```

## 검증 기준

- 장비, FW, 앱, 서버 버전이 모두 기록되었는가
- 재현 절차가 다른 사람이 따라할 수 있는가
- 로그와 캡처가 연결되어 있는가
- 원인 후보가 영역별로 분리되었는가
- 다음 테스트 항목이 명확한가
