# 회로 변경 ECN 하네스

## 목적

회로 변경이 발생했을 때 변경 사유, 영향 범위, BOM, 핀맵, 테스트 항목, 승인자를 남긴다.

## 입력 자료

- 회로도 변경 전후
- PCB 버전
- BOM 변경 목록
- 핀맵과 커넥터 정보
- 변경 사유
- 관련 테스트 결과

## 에이전트 역할

- 변경 전후 차이를 표로 정리한다.
- BOM, 핀맵, 커넥터, 전원, 보호회로 영향 여부를 분리한다.
- HW/FW/SW 영향 범위를 표시한다.
- 테스트해야 할 항목을 체크리스트로 만든다.
- 승인자와 적용 시작 버전을 기록한다.

```md
## 변경 사유
## 변경 전후 비교
## BOM 영향
## 핀맵/커넥터 영향

```

## 사람 승인 게이트

회로 변경 적용, PCB 버전 확정, 부품 대체, 전원/보호회로 변경, 양산 또는 출고 반영은 회로 담당자와 PM이 승인합니다.
