# 자동 검증 게이트 하네스

고급 하네스는 에이전트가 결과를 만든 뒤 "검증까지 했는지" 확인하는 장치를 둡니다. 실제 자동화가 없더라도, 먼저 수동 체크리스트를 파일로 고정하면 팀 운영에 바로 쓸 수 있습니다.

## 검증 게이트 종류

| 게이트 | 적용 업무 | 통과 기준 |
| --- | --- | --- |
| 입력 게이트 | 모든 업무 | 입력 자료와 제외 범위가 명확함 |
| 보안 게이트 | 캡처, DB, 계정 | 민감 정보가 마스킹됨 |
| 품질 게이트 | 개발, 디자인, 문서 | 검증 결과가 남아 있음 |
| 승인 게이트 | 비용, 의료, 대외 제출 | 담당자 확인이 필요 항목에 표시됨 |
| 인계 게이트 | 모든 업무 | 다음 담당자가 볼 파일 경로가 있음 |

## PASS/WARN/FAIL/BLOCKED 기준

### PASS

- 입력 자료가 충분하다.
- 산출물이 지정된 형식으로 만들어졌다.
- 검증 결과가 남아 있다.
- 사람 승인 없이 진행 가능한 범위다.

### WARN

- 산출물은 쓸 수 있지만 확인 필요한 가정이 있다.
- 일부 자료가 부족하지만 진행에는 큰 문제가 없다.
- 민감 정보, 비용, 의료 표현, 운영 반영이 포함되어 담당자 확인이 필요하다.

### FAIL

- 요청한 산출물 형식과 다르다.
- 검증하지 않았는데 완료처럼 작성했다.
- 입력 자료를 잘못 해석했다.
- 기존 기준과 충돌한다.

### BLOCKED

- 필수 입력 자료가 없다.
- 승인 없이는 진행할 수 없는 작업이다.
- 접근 권한이 없다.
- 담당자 결정이 먼저 필요하다.

## 수동 검증 양식

```md
# 검증 결과

## 상태
PASS / WARN / FAIL / BLOCKED

## 확인한 입력 자료
- 

## 만든 산출물
- 

## 검증한 항목
- 

## WARN 또는 FAIL 이유
- 

## 사람 승인 필요
- 

## 다음 액션
- 
```

## 자동화로 확장할 수 있는 예

### 개발

- 테스트 실행
- 린트 실행
- 빌드 확인
- 변경 파일 목록 확인
- 금지 명령 차단

### 디자인/PPT

- 이미지 크기 확인
- 민감 정보 마스킹 확인
- 슬라이드 렌더링 확인
- 텍스트 겹침 확인

### 경영지원

- 금액, 수량, VAT 누락 확인
- 견적 비교 기준 확인
- 결제 승인 필요 항목 표시

### 재활도메인

- 의료적 단정 표현 표시
- 안전 제한 누락 확인
- 전문가 검토 필요 항목 표시

## 에이전트에게 주는 검증 요청문

```md
작업 결과를 바로 완료 처리하지 말고, 먼저 검증 게이트를 작성해라.
상태는 PASS, WARN, FAIL, BLOCKED 중 하나로 표시하고, WARN 이상이면 사람 승인 필요 항목을 별도로 적어라.
```
