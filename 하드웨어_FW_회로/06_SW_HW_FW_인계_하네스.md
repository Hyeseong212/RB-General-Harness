# SW HW FW 인계 하네스

## 목적

하드웨어/FW 변경 사항을 소프트웨어팀이 바로 이해하고 반영할 수 있도록 인계 문서를 만든다.

## 입력 자료

- 변경 요약
- 장비 버전
- FW 버전
- 프로토콜 변경 사항
- 오류 코드와 로그 예시
- 테스트 결과
- 소프트웨어 반영 필요 사항

## 에이전트 역할

- 소프트웨어팀이 알아야 할 정보만 별도로 추린다.
- 프로토콜, 데이터 포맷, 오류 코드, 테스트 조건을 표로 정리한다.
- 앱 또는 서버 변경 필요 여부를 표시한다.
- 미확정 항목을 WARN 또는 BLOCKED로 표시한다.

## 인계 문서 형식

```md
## 변경 요약
## 대상 장비/FW 버전
## SW 영향 범위
## 프로토콜 변경
## 데이터 포맷 변경
## 오류 코드/로그 예시
## 테스트 결과
## SW팀 액션 아이템
## 미확정 항목
```
