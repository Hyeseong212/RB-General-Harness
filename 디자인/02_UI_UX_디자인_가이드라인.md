# UI UX 디자인 가이드라인

## 기본 원칙

업무용 UI는 조용하고 밀도 있게 설계한다.

- 사용자가 해야 할 일을 먼저 보이게 한다.
- 검색, 필터, 목록, 상세, 저장 흐름을 예측 가능하게 만든다.
- 반복 작업에 필요한 정보 밀도를 유지한다.
- 장식보다 읽기 쉬운 구조를 우선한다.
- 권한, 상태, 오류 메시지를 명확히 보여준다.

## 업무용/관리형 화면 기준

업무용 또는 관리형 화면은 다음 구조가 기본이다.

1. 현재 위치
2. 검색 조건
3. 주요 액션
4. 목록 또는 상세 정보
5. 저장/취소/내보내기
6. 상태 메시지

## 컴포넌트 기준

| 요소 | 기준 |
|---|---|
| 버튼 | 명령이 분명해야 한다. 저장, 취소, 검색, 내보내기처럼 행동 중심으로 쓴다. |
| 입력창 | 어떤 값을 넣어야 하는지 라벨과 placeholder를 분리한다. |
| 테이블 | 반복 비교가 쉬워야 한다. 컬럼명, 정렬, 페이지네이션을 명확히 둔다. |
| 모달 | 현재 작업 흐름을 끊어도 되는 경우에만 쓴다. |
| 오류 메시지 | 사용자가 다음에 무엇을 해야 하는지 알려준다. |

## 디자인 QA 체크리스트

- 주요 행동이 첫 화면에서 보이는가
- 검색 조건과 결과가 구분되는가
- 저장 후 피드백이 있는가
- 권한이 낮은 사용자에게 불필요한 버튼이 보이지 않는가
- 긴 텍스트가 잘리지 않는가
- 실제 데이터가 들어가도 레이아웃이 깨지지 않는가
- 모바일 또는 좁은 화면이 필요한 경우 흐름이 유지되는가

## 금지 패턴

- 업무용 화면을 랜딩페이지처럼 만드는 것
- 장식 카드가 너무 많은 화면
- 의미 없는 큰 여백
- 버튼명만 보고 행동을 알 수 없는 경우
- 테이블보다 카드가 더 많아 비교가 어려운 경우
