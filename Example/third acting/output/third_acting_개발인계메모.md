# third acting 개발 인계메모

- 작성일: 2026-05-12
- 단계: third acting
- 산출물: `이승로봇_게임재활_UIUX기획_화면상세_20260512.pptx`
- 목적: 콘텐츠 기획을 실무 개발자가 첫 프로토타입으로 옮길 수 있는 화면정의서 형식으로 변환

## 작업 요약

second acting의 샘플 기획 PPT는 콘텐츠 방향을 보여주는 자료였다. third acting에서는 같은 내용을 실제 개발 인계 관점으로 재구성했다. 1차 UI/UX PPT가 추상적으로 보여 외부 UI/UX 기획·PRD·wireframe·user story 자료를 다운로드한 뒤, 실무형 화면정의서 버전으로 다시 작성했다. 이후 사용자가 “플로우만 있고 화면 배치와 컴포넌트 액션이 없다”고 지적해, `1080x1920` 모바일 해상도 기준의 화면상세 PPTX를 추가 생성했다.

## 개발자가 우선 보면 되는 장표

1. 화면상세 PPTX 2페이지: `1080x1920` 공통 배치 규칙
2. 3페이지: P-01 운동 선택 화면, 카드 배치와 액션
3. 4페이지: P-02 미션 안내 화면, 시작/뒤로 액션
4. 5페이지: P-03 미션 수행 화면, 타이머/반응/쉬어가기/중단 입력 액션
5. 6페이지: P-04A 쉬어가기 화면
6. 7페이지: P-04B 중단 입력 화면, 필드 검증
7. 8페이지: P-05 완료 요약 화면
8. 9페이지: T-01 치료사 세션 상세 화면
9. 10~12페이지: 컴포넌트 액션 매트릭스, 이벤트 payload, 개발 티켓

## MVP 개발 범위 제안

| 구분 | 포함 |
| --- | --- |
| 운동 | 팔 앞으로 뻗기 30초 미션 |
| 환자 화면 | P-01, P-02, P-03, P-04, P-05 |
| 치료사 화면 | T-01 |
| 상태 | READY, ACTIVE, REST, STOPPED, COMPLETE |
| 데이터 | Exercise, Mission, Session, EventLog, SafetyInput |
| 검토 | 운동별 reviewStatus, 센서 입력 가능값, 중단 기준 문구, RA 확인 문구 |

## 아직 확정하면 안 되는 것

- 실제 이승로봇이 어떤 센서 입력을 제공하는지
- 운동 강도, 횟수, 중단 기준
- 환자용 피드백 문구의 최종 문안
- 치료 효과처럼 읽힐 수 있는 표현
- 의료기기/광고 관련 외부 발표 문구

## 다음 액팅으로 넘길 수 있는 작업

- 한 화면씩 더 상세한 와이어프레임 제작
- 화면별 상태 전이표 작성
- 이벤트 로그 JSON 예시 작성
- 개발 티켓 단위로 화면/상태/데이터를 쪼개기
- 재활 전문가 확인 결과를 반영한 문구 업데이트
- Figma 또는 HTML 프로토타입에서 `1080x1920` 실제 간격과 터치 영역 검증
