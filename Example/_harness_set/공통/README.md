# 공통 폴더

모든 부서가 공통으로 사용하는 하네스 기본 구조를 정리한 폴더입니다. 에이전트에게 일을 맡기기 전에 목적, 입력 자료, 산출물, 검증 기준을 먼저 고정합니다.

## 이 폴더를 쓰는 경우

- 요청이 아직 막연해서 작업 범위를 먼저 쪼개야 할 때
- 에이전트에게 줄 입력 자료와 제외 범위를 정해야 할 때
- 부서가 달라도 같은 방식으로 산출물을 남기고 싶을 때
- 결과를 PASS, WARN, FAIL 기준으로 검증하고 싶을 때

## 문서 구성

- `01_MD_하네스_기본구조.md`: 하네스 문서의 기본 골격
- `02_역할_에이전트_운영패턴.md`: Owner, Agent, Reviewer 역할 분리 방식
- `03_검증_게이트_체크리스트.md`: 사람 승인과 검증 기준
- `04_범용_산출물_템플릿.md`: 결과물을 남기는 기본 형식

## 공통 하네스 기본 흐름

1. 작업 목적을 한 문장으로 적는다.
2. 입력 자료를 파일 경로, 캡처, 회의록, 데이터 기준으로 나열한다.
3. 이번 작업에서 하지 않을 범위를 분명히 적는다.
4. 에이전트가 만들어야 할 산출물을 지정한다.
5. 검증 기준과 사람 승인 게이트를 둔다.

## 좋은 산출물 기준

- 다음 담당자가 바로 이어받을 수 있다.
- 판단 근거와 가정이 분리되어 있다.
- 누락, 위험, 검증 불가 항목이 숨겨져 있지 않다.
- 결과가 말로만 끝나지 않고 파일로 남아 있다.
