# 검증 게이트 체크리스트

## 목적

검증 게이트는 에이전트 산출물을 바로 사용하지 않고, 사람이 믿고 검토할 수 있는 상태로 만들기 위한 기준이다.

## 공통 검증 항목

| 항목 | 확인 질문 |
|---|---|
| 목적 적합성 | 산출물이 처음 목표와 맞는가 |
| 입력 충실도 | 제공된 자료를 빠뜨리지 않았는가 |
| 근거 구분 | 사실, 추정, 의견이 구분되어 있는가 |
| 누락 확인 | 빠진 이해관계자, 조건, 예외가 없는가 |
| 형식 일관성 | 정해진 출력 형식을 지켰는가 |
| 재사용성 | 다음 사람이 이어받을 수 있는가 |
| 위험 표시 | 사람이 승인해야 할 부분이 표시됐는가 |

## 개발 검증

- 빌드 또는 실행 확인
- 테스트 또는 재현 절차 확인
- 변경 범위 확인
- 기존 기능 영향 확인
- 보안/데이터 영향 확인

## 기획 검증

- 요구사항과 산출물이 연결되는가
- 담당자와 기한이 있는가
- 일정 의존관계가 보이는가
- 리스크와 보류사항이 분리되어 있는가
- 외부 제출 문구는 사람이 검토했는가

## 디자인 검증

- 사용자가 해야 할 행동이 명확한가
- 정보 우선순위가 보이는가
- 실제 데이터가 들어가도 깨지지 않는가
- 접근성, 가독성, 상태 표현이 충분한가
- 발표자료라면 한 장 한 메시지가 지켜졌는가

## 승인 상태

산출물 끝에는 아래 중 하나를 남긴다.

```text
상태: PASS
설명: 기준 충족. 다음 단계로 전달 가능.

상태: WARN
설명: 사용은 가능하지만 확인이 필요한 항목이 있음.

상태: BLOCKED
설명: 입력 부족 또는 검증 실패로 다음 단계 진행 불가.
```
