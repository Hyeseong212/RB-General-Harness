# 폴더별 문서 정리

이 저장소는 회사에서 에이전트를 막연히 쓰는 것이 아니라, 부서별 반복 업무를 하네스 MD로 고정해서 사용하기 위한 가이드라인 모음입니다.

하네스는 한 번 쓰고 버리는 프롬프트가 아니라, 같은 유형의 업무를 계속 맡길 수 있도록 만든 작업 계약서입니다. 각 폴더는 담당 부서가 바로 열어보고 쓸 수 있도록 목적, 사용 상황, 입력 자료, 산출물, 검증 기준을 따로 정리합니다.

## 전체 폴더 구조

| 폴더 | 담당 영역 | 주요 사용자 | 목적 |
| --- | --- | --- | --- |
| `공통/` | 모든 부서 공통 | 전 직원 | Task brief, 검증 게이트, 산출물 양식 |
| `기획/` | 기획, PM, 문서화 | 기획자, PM | 회의록, 요구사항, PPT 초안, 제안서 정리 |
| `개발/` | 소프트웨어 개발 | 개발자 | 코드 분석, 구현, QA, 서브 에이전트 운영 |
| `디자인/` | UI/UX, PPT 디자인 | 디자인 담당, 기획자 | 화면 캡처, UI/UX 검토, PPT 스타일 정리 |
| `하드웨어_FW_회로/` | 하드웨어, F/W, 회로 | 조립, F/W, 회로 담당 | 조립 QC, FW 릴리즈, 회로 ECN, 통합테스트, 소프트웨어 인계 |
| `경영지원/` | 구매, 외주, 과제비 | 경영지원, PM | 견적 비교, 과제비 증빙, 외주 계약 검토, 일정 리스크 |
| `재활도메인/` | 재활 전문 검토 | 재활 박사님, PM | 훈련 지표, 환자 문구, 안전 제한, 임상자문, 근거 확인 |
| `docs/` | 회사 운영 문서 | PM, 발표 담당 | 회사 역할별 에이전트 활용 맵 |
| `templates/` | 템플릿 | 전 직원 | 반복 Task 작성용 기본 양식 |

## 기본 사용 순서

1. `공통/README.md`를 보고 Task brief를 짧게 만든다.
2. 작업 성격에 맞는 부서 폴더로 이동한다.
3. 해당 폴더의 `README.md`에서 사용 상황과 산출물 기준을 확인한다.
4. 에이전트에게 입력 자료, 제외 범위, 산출물 위치를 명확히 준다.
5. 결과를 PASS, WARN, FAIL로 검증하고 사람 승인 게이트를 통과시킨다.

## 사람 승인이 필요한 항목

- 환자 또는 계정 식별 정보가 포함된 자료
- 운영 DB, 실서비스 데이터, 장비 출고와 관련된 변경
- 대외 제출 문구, 계약 문구, 비용 집행
- 의료적 판단, 치료 효과, 안전 제한과 관련된 표현
- 제품 성능을 단정하거나 외부에 공개되는 문장

## 발표자료에서 강조할 문장

에이전트 하네싱의 핵심은 "AI에게 질문을 잘하는 방법"이 아니라 "AI 답변이 회사 산출물로 남도록 업무 구조를 만드는 방법"입니다.
