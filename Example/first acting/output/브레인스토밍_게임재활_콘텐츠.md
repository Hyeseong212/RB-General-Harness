# 브레인스토밍: 간단한 재활운동을 게임운동 콘텐츠로 바꾸기

- 작성일: 2026-05-12
- 목적: 이승로봇 콘텐츠 기획 PPT에서 사용할 "재활운동의 게임화" 방향을 잡는다.
- 위치: first acting 자료조사/브레인스토밍 결과
- 정정 메모: 앞선 자료조사는 안전 표현과 근거 정리에 치우쳤다. 이 문서는 PPT의 실제 중심 주제인 "간단한 재활운동을 게임운동으로 승화시키는 콘텐츠"에 맞춰 방향을 다시 잡은 브레인스토밍 결과다.

## 한 문장 방향

간단한 재활운동을 "반복해야 하는 숙제"가 아니라, 목표·반응·보상·기록이 있는 짧은 게임 루프로 바꾼다.

## 핵심 메시지

자료조사 에이전트는 PPT를 바로 만드는 사람이 아니다.  
다음 에이전트가 믿고 쓸 수 있도록, 게임화 아이디어와 안전 표현, 위험 표현, 출처, 전문가 확인 질문을 분리해 주는 역할이다.

## 콘텐츠 기획 중심축

| 축 | 설명 | PPT에서 쓸 문장 |
| --- | --- | --- |
| 운동의 재해석 | 단순 반복 동작을 게임 목표로 바꾼다. | "팔을 뻗는 동작은 과제를 수행하는 미션이 된다." |
| 즉시 피드백 | 움직임이 화면 반응으로 돌아온다. | "움직임이 보이면 반복은 지루함보다 몰입에 가까워진다." |
| 작은 성공 | 큰 회복 약속 대신 오늘의 성공을 보여준다. | "콘텐츠는 회복을 단정하지 않고, 오늘의 수행을 확인한다." |
| 기록 | 세션 결과를 다음 훈련의 재료로 남긴다. | "게임 결과는 치료사가 참고할 수 있는 관찰 기록이 된다." |
| 안전 제한 | 게임이 무리한 반복을 유도하지 않게 한다. | "재미보다 먼저 멈춤 기준과 전문가 확인이 필요하다." |

## 운동을 게임으로 바꾸는 기본 루프

```text
목표 제시
→ 동작 수행
→ 화면 반응
→ 점수/진행률 표시
→ 휴식 또는 안전 체크
→ 기록 저장
→ 다음 난이도 또는 다음 미션
```

## 아이디어 목록

| 기본 재활운동 | 게임운동 콘셉트 | 화면 피드백 | 기록할 수 있는 값 | 주의 |
| --- | --- | --- | --- | --- |
| 팔 앞으로 뻗기 | 과일 따기 / 버튼 밝히기 | 목표 지점에 닿으면 빛이 켜짐 | 도달 횟수, 좌우 차이, 중단 사유 | 어깨 통증 유발 표현 주의 |
| 손 쥐었다 펴기 | 보석 모으기 / 문 열기 | 손 동작마다 아이템 획득 | 반복 횟수, 속도, 피로도 | 세밀한 기능 향상 단정 금지 |
| 앉아서 무릎 펴기 | 풍선 터뜨리기 / 발끝으로 타일 누르기 | 다리 움직임에 따라 타일 반응 | 반복 횟수, 수행 시간 | 통증·피로 시 중단 안내 |
| 체중 좌우 이동 | 균형 게이지 맞추기 | 중심이 범위 안에 들어오면 게이지 유지 | 유지 시간, 흔들림, 보조 필요 여부 | 낙상 위험 표현과 보호자/치료사 감독 필요 |
| 제자리 걷기 | 리듬 타일 밟기 | 박자에 맞으면 콤보 표시 | 스텝 수, 리듬 유지, 휴식 횟수 | 보행 회복 표현 금지 |
| 자세 펴기 | 에너지 충전 / 빛 밝히기 | 자세 유지 시간만큼 화면이 밝아짐 | 유지 시간, 휴식 간격 | 자세 교정 효과 단정 금지 |
| 호흡/이완 | 파동 맞추기 / 불빛 안정화 | 호흡 리듬에 맞춰 화면 변화 | 세션 시간, 자가 피로도 | 치료 효과처럼 표현 금지 |

## 사용자별 콘텐츠 관점

| 사용자 | 알고 싶은 것 | 게임화 포인트 | 안전한 표현 |
| --- | --- | --- | --- |
| 환자 | 오늘 무엇을 하면 되는지 | 짧은 미션, 즉시 반응, 쉬운 성공 | "오늘의 미션을 하나씩 수행합니다." |
| 보호자 | 어디까지 도와야 하는지 | 관찰 메모, 피로/통증 체크 | "훈련 전후 변화를 기록해 상담에 참고합니다." |
| 치료사 | 실제로 어떤 움직임이 나왔는지 | 반복 수, 중단 사유, 좌우 차이 | "수행 기록은 치료사의 판단을 돕는 참고 자료입니다." |
| 관리자 | 콘텐츠가 운영 가능한지 | 세션 기록, 사용 현황, 안전 체크 | "운영 흐름과 기록 기준을 표준화합니다." |

## PPT에 넣을 수 있는 장표 방향

### 슬라이드 A. 재활운동을 게임 루프로 바꾸다

- 핵심 문장: "운동은 그대로인데, 경험이 달라진다."
- 보여줄 구조: 재활운동 → 미션 → 피드백 → 기록 → 다음 미션
- 장표 톤: 실행 결과가 아니라 운영 예시/기획 방향

### 슬라이드 B. first acting은 게임 아이디어와 안전 기준을 분리한다

- 먼저 읽는 하네스: 브레인스토밍, 리서치 의사결정, 팩트체킹, 환자문구 안전제한, 재활근거 팩트체킹
- 산출물: 게임화 아이디어, 안전 문장, 위험 문장, 출처, 전문가 확인 질문
- 핵심 문장: "재미있는 문장과 안전한 문장은 따로 검토해야 한다."

### 슬라이드 C. second acting은 검증된 재료만 장표화한다

- 입력값: first acting 산출물
- 하는 일: PPT 구조화, 메시지 압축, 시각화
- 하지 않는 일: 치료 효과 단정, 제품 성능 과장, 출처 없는 주장 사용
- 핵심 문장: "자료조사 결과는 최종 문서가 아니라 다음 작업자를 위한 검증된 입력값이다."

### 슬라이드 D. 왜 표현을 분리해야 하는가

- 안전한 문장: "반복 수행을 돕습니다."
- 위험한 문장: "기능이 회복됩니다."
- 출처: 근거가 있는 범위만 사용
- 전문가 확인 질문: 환자 안전과 운동 강도는 사람이 판단
- 핵심 문장: "게임처럼 보여도, 재활 콘텐츠는 안전 기준을 먼저 가져야 한다."

## 안전한 문장 후보

- "간단한 재활운동을 짧은 미션과 피드백이 있는 게임 경험으로 바꿉니다."
- "반복 수행을 더 이해하기 쉽고 참여하기 쉬운 형태로 설계합니다."
- "훈련 기록은 치료사가 상태 변화를 관찰하는 참고 자료가 될 수 있습니다."
- "게임 요소는 동기를 돕는 장치이며, 운동 강도와 대상자는 전문가 확인이 필요합니다."
- "콘텐츠는 회복을 단정하지 않고, 수행 과정과 기록을 보여줍니다."

## 위험한 문장 후보

| 위험 문장 | 이유 | 수정 방향 |
| --- | --- | --- |
| "게임을 하면 재활 효과가 올라갑니다." | 치료 효과 단정 | "게임 요소는 반복 참여를 돕는 방식으로 설계할 수 있습니다." |
| "환자가 스스로 안전하게 훈련할 수 있습니다." | 감독/환경/대상자 조건 누락 | "사용 환경과 대상자는 전문가 확인이 필요합니다." |
| "기능이 회복됩니다." | 결과 보장 | "수행 변화와 기록을 확인할 수 있습니다." |
| "재미있어서 계속 하게 됩니다." | 무리한 반복 유도 가능 | "휴식과 중단 기준을 포함해 참여 경험을 설계합니다." |
| "로봇이 맞춤형 치료를 제공합니다." | 의료 판단/제품 효과 주장 | "설정된 기준에 따라 훈련 경험과 참고 정보를 제공합니다." |

## 바로 실험할 수 있는 작은 Task

1. 팔 뻗기 운동 하나를 골라 30초짜리 미션 화면으로 바꾼다.
2. "성공/실패" 대신 "도달/휴식/다시 시도"로 피드백 문구를 만든다.
3. 환자용 화면과 치료사용 기록 화면을 분리해 본다.
4. 게임 보상 문구 10개를 만들고, 무리한 반복을 유도하는 문장을 WARN 처리한다.
5. 같은 운동을 보호자 설명용, 치료사 설명용, 관리자 설명용으로 각각 한 문장씩 바꾼다.

## 전문가 확인 필요

- 어떤 운동을 게임화해도 되는지
- 환자에게 보여줘도 되는 지표와 숨겨야 할 지표
- 통증, 피로, 어지러움, 균형 문제 발생 시 중단 문구
- 게임 보상이 무리한 반복을 유도하지 않는지
- "재활운동", "운동게임", "훈련 콘텐츠" 중 어떤 용어가 가장 안전한지
