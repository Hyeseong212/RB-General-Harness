## 슬라이드별 맥락 점검

| 페이지 | 역할 | 맥락 연결 판단 | 메모 |
| --- | --- | --- | --- |
| 1 | 핵심 제안 | PASS | 단순 운동을 게임운동 경험으로 바꾸는 전체 방향을 제시함 |
| 2 | 문제 정의 | PASS | 반복운동 자체보다 `왜 계속해야 하는지`가 보이지 않는 문제를 제시함 |
| 3 | 기획 원리 | PASS | 2장의 문제를 `반복 루프 설계`라는 해법으로 받음 |
| 4 | 콘텐츠 예시 | PASS | 3장의 루프 원리를 실제 운동·미션·반응·기록 예시로 변환함 |
| 5 | 환자 화면 | PASS | 사용자에게 먼저 보여야 할 정보와 안전 입력을 제시함 |
| 6 | 치료사 화면 | PASS | 게임 결과를 점수 자랑이 아니라 관찰 기록으로 정리함 |
| 7 | 운영 관점 | PASS | 보호자·관리자·재활 전문가가 각자 봐야 할 흐름을 분리함 |
| 8 | 표현 기준 | PASS | 재미있는 문장과 위험한 문장을 분리해야 하는 이유를 정리함 |
| 9 | 하네싱 구조 | PASS | first acting 자료조사와 second acting PPT 제작 역할을 분리함 |
| 10 | 다음 단계 | PASS | 다음 실험을 30초 미션 프로토타입으로 좁혀 실행 단위를 제시함 |

## 흐름 판단

- 전체 흐름은 `제안 → 문제 → 원리 → 예시 → 사용자별 화면 → 안전한 표현 → 제작 인계 → 다음 실험` 순서로 이어진다.
- 8~9장은 콘텐츠 아이디어 자체보다 운영·표현 기준을 다루므로, 투자/홍보용 발표에서는 부록으로 내려도 된다.
- 현재 목적이 기획 PPT라면 8~9장을 본문에 두는 편이 안전하다. 재활 콘텐츠는 재미보다 안전한 표현과 전문가 확인 구조가 중요하기 때문이다.
