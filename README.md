# Agent Harnessing Guidelines

이 저장소는 회사 내부에서 AI 에이전트를 개인 프롬프트가 아니라 반복 가능한 업무 산출물 구조로 쓰기 위한 기준 문서 모음이다.

## 목적

- 개인별 AI 사용법을 통일하려는 문서가 아니다.
- 개발, 디자인, 기획, QA, 문서화 업무를 Task 단위로 나누고 입력/출력/검증 기준을 남기는 것이 목적이다.
- AI 산출물은 바로 정답으로 쓰지 않고, 사람이 검토 가능한 중간 산출물로 취급한다.

## 현재 회사 구성 기준

- 기획자 1명
- 소프트웨어 개발자 6명
- 하드웨어 조립 1명
- F/W 2명
- 회로 1명
- 경영지원 1명
- PM 1명
- 재활박사 1명

## 기본 원칙

1. 한 번에 큰 일을 맡기지 않고 Task로 쪼갠다.
2. 각 Task에는 입력 자료, 담당 역할, 기대 산출물, 검증 기준을 적는다.
3. 구현 에이전트와 검증 에이전트를 분리한다.
4. 개인정보, 민감정보, 운영 DB 변경, 법무/규제/안전 관련 판단은 사람 승인 없이 확정하지 않는다.
5. 결과는 문서, 코드, 표, 테스트 결과, 리스크 목록처럼 다음 사람이 이어받을 수 있는 형태로 남긴다.
