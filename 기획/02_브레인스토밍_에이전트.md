# 브레인스토밍 에이전트

## 목적

브레인스토밍 에이전트는 초기에 가능한 선택지를 넓게 뽑는 역할이다.  
정답을 고르는 역할이 아니라, 빠진 가능성을 줄이는 역할이다.

## 사용하기 좋은 상황

- 새 과제 제안서를 준비할 때
- 기능 후보를 넓게 보고 싶을 때
- 부서별 AI 활용 후보를 찾을 때
- 회의 전에 안건을 정리할 때
- 기존 문제의 원인을 여러 관점에서 보고 싶을 때

## 입력 자료

- 주제
- 현재 문제
- 대상 사용자 또는 부서
- 제약 조건
- 반드시 피해야 할 방향
- 참고 자료

## 출력 형식

```text
1. 아이디어 목록
2. 부서별 적용 가능성
3. 기대 효과
4. 예상 리스크
5. 바로 실험할 수 있는 작은 Task
```

## 요청 예시

```text
의료기기 국가과제 업무에서 AI 에이전트를 적용할 수 있는 반복 업무를 부서별로 찾아줘.
부서는 소프트웨어, 펌웨어, 하드웨어, 회로, 기획, 경영지원, 재활의학이다.
각 부서별로 바로 실험 가능한 작은 Task 3개씩 제안해줘.
```

## 주의점

브레인스토밍 결과는 그대로 실행하지 않는다.  
반드시 정리 에이전트나 사람이 우선순위와 실행 가능성을 다시 검토한다.
