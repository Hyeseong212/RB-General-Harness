# HW/FW/회로 변경 영향 하네스

## 목적

하드웨어, F/W, 회로 변경이 제품 동작, 앱, 서버, 데이터, 테스트에 미치는 영향을 빠르게 정리한다.

## 입력 자료

- 변경 전후 사진 또는 도면
- 회로도, 핀맵, 배선 정보
- F/W 버전, 프로토콜 문서, 로그
- 테스트 조건, 재현 절차, 실패 증상
- 소프트웨어팀에 전달해야 할 제약 조건

## 에이전트 역할

- 변경 내용을 항목별로 분해한다.
- 영향 범위를 하드웨어, F/W, 앱, 서버, 데이터로 나눈다.
- 테스트해야 할 항목을 체크리스트로 만든다.
- 사람이 확인해야 할 위험 항목을 WARN으로 표시한다.

## 산출물 형식

```md
## 변경 요약
## 영향 범위
## 버전 및 대상 장비
## 테스트 체크리스트
## 소프트웨어 인계사항
## WARN 항목
## 담당자 확인 필요
```

## 영향 범위 표

| 영역 | 영향 내용 | 확인 방법 | 담당자 | 상태 |
| --- | --- | --- | --- | --- |
| HW |  | 육안/측정/조립 확인 |  | PASS/WARN/FAIL |
| FW |  | 버전/로그/통신 확인 |  | PASS/WARN/FAIL |
| 회로 |  | 회로도/핀맵/전원 확인 |  | PASS/WARN/FAIL |
| 앱 |  | 화면/동작/오류 확인 |  | PASS/WARN/FAIL |
| 서버 |  | API/DB/로그 확인 |  | PASS/WARN/FAIL |

## 작업 절차

1. 변경 전후 버전과 사진을 정리한다.
2. 변경 사유를 한 문장으로 적는다.
3. 영향 범위를 HW, FW, 회로, 앱, 서버로 나눈다.
4. 실제 장비 테스트가 필요한 항목을 표시한다.
5. 소프트웨어팀 인계사항을 프로토콜, 로그, 오류 코드 기준으로 작성한다.
6. 담당자 승인 필요 항목을 WARN 또는 BLOCKED로 표시한다.

## 금지 사항

- 실제 장비 테스트 없이 정상 동작을 확정하지 않는다.
- 안전, 전원, 회로 보호, 의료기기 동작 관련 판단을 단정하지 않는다.
- 담당자 승인 없이 배포 또는 출고 결정을 내리지 않는다.
