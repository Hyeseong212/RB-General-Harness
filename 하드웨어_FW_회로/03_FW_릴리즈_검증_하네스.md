# FW 릴리즈 검증 하네스

## 목적

펌웨어 릴리즈 전후에 버전, 변경 내용, 프로토콜, 로그, 앱/서버 영향, 롤백 조건을 정리한다.

## 입력 자료

- 펌웨어 버전
- 변경 내역
- 빌드 파일 또는 배포 파일명
- 프로토콜 문서
- 오류 코드 목록
- 테스트 로그
- 대상 장비 버전

## 에이전트 역할

- 릴리즈 노트를 표준 형식으로 정리한다.
- 앱과 서버가 알아야 할 프로토콜 변경을 분리한다.
- 오류 코드와 로그 예시를 정리한다.
- 테스트 항목과 미검증 항목을 표시한다.
- 롤백 또는 재배포 조건을 WARN으로 표시한다.

## 릴리즈 노트 형식

```md
## FW 버전
## 대상 장비
## 변경 요약
## 프로토콜 변경
## 오류 코드 변경
## 테스트 결과
## 미검증 항목
## 롤백 조건
## SW팀 인계사항
```

## 검증 체크리스트

| 항목 | 확인 |
| --- | --- |
| FW 버전과 빌드 파일명이 일치하는가 | 예/아니오 |
| 대상 장비 버전이 명시되어 있는가 | 예/아니오 |
| 프로토콜 변경이 앱/서버에 전달되었는가 | 예/아니오 |
| 오류 코드와 로그 예시가 있는가 | 예/아니오 |
| 실제 장비 테스트 결과가 있는가 | 예/아니오 |
| 롤백 조건이 정리되어 있는가 | 예/아니오 |

## 사람 승인 게이트

펌웨어 배포, 장비 출고 반영, 프로토콜 변경 확정, 안전 동작 변경은 F/W 담당자와 PM이 승인합니다.
