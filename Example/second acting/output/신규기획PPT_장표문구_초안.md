# 신규 기획 PPT 장표 문구 초안

- 작성일: 2026-05-12
- 대상 PPT: 이승로봇 게임재활 콘텐츠 기획 PPT
- 정정 메모: 기존 발표자료에 삽입할 문구가 아니라, 별도 신규 기획 PPT에 사용할 문구 초안이다.

## 1페이지

**제목**

운동은 그대로인데, 경험이 달라진다

**서브 문장**

간단한 재활운동을 짧은 미션과 피드백이 있는 게임운동 콘텐츠로 바꿉니다.

## 2페이지

**제목**

반복운동에는 "왜 계속해야 하는지"를 보여주는 장치가 필요하다

**본문 후보**

- 단순 반복은 참여 동기가 약해질 수 있다.
- 환자는 수행 상태를 즉시 이해하기 어렵다.
- 치료사와 보호자는 수행 과정과 중단 이유를 보고 싶다.

## 3페이지

**제목**

게임화의 핵심은 재미보다 반복 루프다

**본문 후보**

목표 제시 → 동작 수행 → 화면 반응 → 진행률 → 휴식 체크 → 기록 저장 → 다음 미션

**강조 문장**

게임 요소는 운동을 대신하지 않고, 반복 수행을 이해하기 쉬운 경험으로 바꾼다.

## 4페이지

**제목**

기본 운동은 작은 미션으로 바뀐다

**본문 후보**

| 운동 | 게임 미션 | 기록 |
| --- | --- | --- |
| 팔 뻗기 | 과일 따기 | 도달 횟수 |
| 손 쥐기 | 보석 모으기 | 반복 횟수 |
| 무릎 펴기 | 풍선 터뜨리기 | 수행 시간 |
| 체중 이동 | 균형 게이지 | 유지 시간 |

## 5페이지

**제목**

환자에게는 오늘의 미션이 먼저 보여야 한다

**본문 후보**

- 오늘의 미션
- 남은 횟수
- 쉬어가기
- 다시 시도
- 통증/피로 입력

**안전 문장**

"오늘의 미션을 하나씩 수행합니다."

## 6페이지

**제목**

치료사에게는 게임 결과가 관찰 기록이 된다

**본문 후보**

- 반복 횟수
- 수행 시간
- 중단 사유
- 좌우 차이
- 통증/피로 메모

**안전 문장**

"수행 기록은 치료사의 판단을 돕는 참고 자료입니다."

## 7페이지

**제목**

보호자와 관리자는 재미보다 안전한 운영 흐름을 본다

**본문 후보**

- 보호자: 피로, 통증, 훈련 전후 변화 메모
- 관리자: 세션 수, 사용 현황, 안전 체크
- 재활 전문가: 대상자, 운동 범위, 중단 기준 검토

## 8페이지

**제목**

재미있는 문장과 안전한 문장은 따로 검토해야 한다

**본문 후보**

| 안전한 문장 | 위험한 문장 |
| --- | --- |
| "반복 수행을 이해하기 쉬운 형태로 설계합니다." | "게임을 하면 재활 효과가 올라갑니다." |
| "수행 기록을 참고 자료로 남깁니다." | "기능이 회복됩니다." |

**강조 문장**

재미는 참여를 돕는 장치이고, 효과와 안전은 검증해야 할 영역이다.

## 9페이지

**제목**

자료조사 에이전트는 PPT를 만드는 사람이 아니라 기준을 정리하는 사람이다

**본문 후보**

- first acting: 아이디어, 출처, 안전 문장, 위험 문장, 전문가 질문
- second acting: 장표 구조, 메시지 압축, 시각화 방향
- 다음 단계: UI/UX 흐름 또는 프로토타입 제작

## 10페이지

**제목**

다음 실험은 한 가지 운동을 30초 미션으로 만드는 것이다

**본문 후보**

1. 팔 뻗기 운동 선택
2. 30초 미션 화면 구성
3. 도달/휴식/다시 시도 피드백 작성
4. 환자 화면과 치료사 기록 화면 분리
5. 재활 전문가가 중단 기준 검토

## 사용 금지 문구

- "게임으로 재활 효과가 올라갑니다."
- "기능이 회복됩니다."
- "환자가 혼자 안전하게 사용할 수 있습니다."
- "로봇이 맞춤형 치료를 제공합니다."
- "검증된 치료 콘텐츠입니다."
